CodeRanch Dibs Reservation System

Every few months, The CodeRanch administrators select candidates for Forum Moderator and through the forums open for bidding.

Historically, this process has been done via repeated updates of a message thread. But that's hard to track in real time, so I've developed this app. In addition to its ability to do useful work, it's also my sandbox/showcase for the RichFaces Version 4 JSF extension tag collection and the Neo4j Nosql DBMS.

User Roles

This app, when properly secured, supports 2 types of users.

The Dibs User interacts with the Dibs Selection screen. This is a fairly simple screen with a shuttle control on it. Users place Dibs by shuttling forums of interest to the "selected" side of the control.

The Dibs Administrator has access to additional functions:

  • Reports and archiving the database to YAML-format text files.
  • Clearing and loading the database from YAML
  • Manual selection of a user ID to make Dibs selections for that user.
